yFinance API

API REST para consulta de métricas financieras de tickers USA y Chile (IPSA), con normalización FX y cálculo de indicadores de riesgo/retorno.

URL Base
https://yfinance-api-prod.up.railway.app
Versión
1.0.0
Fuente
Yahoo Finance / yfinance ≥0.2.x
Estado
● Online

Introducción

Esta API permite consultar métricas fundamentales y estadísticas para carteras mixtas de activos financieros. Soporta ETFs y acciones de mercados USA (NYSE, NASDAQ) y Chile (Bolsa de Santiago, sufijo .SN).

Los activos chilenos se convierten automáticamente a USD antes de calcular métricas, permitiendo comparar directamente activos de ambos mercados.

POST /historia Serie OHLCV diaria para un ticker

Devuelve la serie histórica OHLCV diaria para un único ticker. Los precios están ajustados por dividendos y splits. Útil para generación de gráficos de velas o línea. Resultado cacheado 15 minutos.

Request Body

{
  "ticker":  "SCHD",
  "mercado": "USA",   // "USA" (default) o "CHILE"
  "periodo": "1mo"    // default: "1mo"
}

Parámetros

CampoTipoRequeridoDescripción
tickerstringSíSímbolo del activo
mercadostringNo"USA" (default) o "CHILE"
periodostringNoPeríodo histórico. Default: "1mo". Mismos valores que /metricas

{

  "ticker":  "SCHD",
  "periodo": "1mo",
  "data": [

      "open":   31.80,
      "high":   32.10,

      "close":  32.00,
      "volume": 18500000
    },

CampoDescripción

Nota: Para activos chilenos (mercado: "CHILE"), los precios se devuelven en CLP — sin conversión a USD. Este endpoint entrega los datos crudos para graficar.

POST /correlacion Matriz de correlación de retornos

Devuelve la matriz de correlación de retornos logarítmicos diarios entre todos los tickers, normalizados a USD. Resultado cacheado 15 minutos.

Request body: mismo formato que /metricas.

}

0.0 – 0.5Baja correlación — buena diversificación
< 0.0Correlación negativa — diversificación óptima

Restricciones y limitaciones

Yahoo Finance (fuente de datos)

⚡

Rate Limiting

No documentado oficialmente. Exceder ~2.000 requests/hora por IP puede resultar en bloqueo temporal (HTTP 429 o 401).

🔑

Sin API Key oficial

yfinance hace scraping de Yahoo Finance. No hay SLA ni garantía de disponibilidad continua.

🕐

Datos con retraso

Precios en tiempo real solo para USA en horario de mercado. Chile (S&P CLX) puede tener retraso de 15–20 min.

📉

Tickers deslistados

.info puede devolver {} o lanzar excepción para activos suspendidos o delisted.

🇨🇱

Cobertura IPSA

No todos los activos del IPSA tienen datos completos. Activos con baja liquidez pueden tener gaps en la serie histórica.

null

Campos nulos

trailingPE, forwardPE pueden ser null para ETFs, fondos o activos sin earnings reportados.

Esta API

RestricciónDetalle
Máx. 20 tickers / requestLímite de diseño para evitar timeouts. Cada ticker agrega ~0.2s de espera por rate limiting
Caché 15 minutosDos requests idénticos en 15 min devuelven el mismo dato. El campo source indica si es "cache" o "live"
Caché volátilVive en memoria del proceso. Un reinicio del servidor (deploy, crash) la borra completamente
Sin datos intradayPipeline usa interval="1d". No apto para estrategias intraday
Tasa libre fijaSharpe calculado con rf=4.5% hardcodeada en el servidor. No se actualiza automáticamente
Latencia variableUn request de 20 tickers en frío puede tardar 15–25 segundos. Con caché, respuesta inmediata

Integración con sistema de alertas

Campo source: Usar "source": "cache" para saber si el dato tiene hasta 15 min de antigüedad. Para alertas críticas, esperar la próxima ventana "live".
Polling eficiente: Con frecuencia menor a 15 min, el segundo request siempre es cache — no consume quota de Yahoo Finance.
Tickers fallidos: Si un ticker no aparece en data, verificar siempre tickers_procesados vs cantidad enviada. Ausencia silenciosa indica fallo de red, delisting o ticker inválido.
Sharpe negativo: sharpe_ratio < 0 (como COPEC.SN = -0.10) indica retorno histórico inferior a la tasa libre de riesgo — señal directa para el motor de alertas.
Correlación alta: Pares con correlación > 0.85 son candidatos a reducir posición en uno — la cartera no se beneficia de tener ambos.
